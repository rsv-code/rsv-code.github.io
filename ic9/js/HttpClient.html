postBinary(PostTypeStr, Obj, ContentType)

Performs a HTTP POST call and returns the results of the call as a Buffer object. The result object sets the response body as the 'content' property of the result object.

URL_ENCODED: When post type is set as url_encoded it is expected this the provided object is a JS object with properties set as key/value pairs to be used as the post parameters. In this case ContentType argument is ignored.

MULTIPART: When post type is set as multipart it is expected this the provided object is an object/map where the values are HttpPart JS objects. (See net/HttpPart.js for details.) For each part provided, only the 'name' and 'data' properties need to be set, and the 'data' property is expected to be a Java byte[]. In this case the ContentType argument is ignored.

CUSTOM: When the post type is set at custom it is expected this the provided object is either a plain String or a JS Buffer object. The Obj String or Buffer object is written as the body of the request and the ContentType argument must be set. Use this for example when you want to post JSON data. In this case you'd set the Obj as the JSON encoded string and then set ContentType to 'application/json'.

Obj Is an Object. (See above for details.)
